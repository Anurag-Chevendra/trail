Trail

~30s to a verdict

Suspicious npm packages, installed in a disposable sandbox and watched at the syscall level. Not what the code says it does but rather what it actually did.

What this is

Trail is a dynamic-analysis pipeline for npm packages. Each package on the feed is installed inside a fresh, throwaway container while every file access, network connection, and spawned process is recorded. The trace is then parsed and each action is attributed to the process that made it, so the package's own behavior is separated from the normal noise of an npm install. Anomalous behavior surfaces in about 30 seconds on average. Any lag you see on the feed is queue time, not analysis time. Throughput is currently limited by available RAM.

It is a triage tool, not a malware classifier. Its job is to surface behavior a human should look at, not to pass final judgment.

How a package moves through it

  1. Fresh sandboxEvery package starts from an identical, pristine container image. Nothing persists between runs.
  2. Instrumented installThe install runs with scripts enabled, under full syscall tracing, isolated from any real credentials or user data.
  3. AttributionEach recorded action is tied back to the process tree that produced it, package code, npm itself, or build tooling.
  4. VerdictParsers turn the trace into a structured report; anything notable marks the package for human review on the feed.

What gets flagged

unexpected network callbacks known exfiltration services credential-path probes dropped & executed payloads filesystem reconnaissance temp-file snooping files written outside the project

The feed has already surfaced live threats, including dependency-confusion probes beaconing to out-of-band callback services and a package exfiltrating host details encoded inside DNS queries.

Reading the feed

REVIEW

Something in the trace warrants a human look. Not a conviction, a lead.

CLEAN

Nothing notable observed during install. Behavior outside the install window is out of scope.

INCONCLUSIVE

The install stalled or was cut short, so a clean-looking trace can't be trusted.

If the registry pulls a package while it is being analyzed, it drops off the feed entirely. Security holdings are the exception: npm replaces the real version with a 0.0.1-security placeholder, so there is nothing left to install and nothing to trace. Those stay, filed under review and marked as placeholders.

Engine

A sandbox is table stakes. The open problem is detection strategy, and it is worked on across two branches in parallel.
